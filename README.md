# Optimizing-IT-Support-Team-Performance

**Problem Statement:**

The IT support team receives many tickets every day, and sometimes tickets take longer to resolve. 
-Some agents may have more workload, and some issues may repeat frequently. Because of this, customer satisfaction may decrease. 
-This project aims to analyze the ticket data to understand team performance, find problem areas, and suggest ways to improve response time and service quality.

__Dataset Description:__


The dataset contains detailed IT support ticket records used to analyze and optimize support team performance. Each record represents a single customer support interaction and includes customer demographics, product details, and ticket handling information.

The dataset includes the following key attributes:

__Ticket ID –__ Unique identifier for each support ticket

__Customer Name, Email, Age, Gender –__ Customer demographic details

__Product Purchased & Date of Purchase –__ Information about the product related to the issue

__Ticket Type, Subject, Description –__ Nature and details of the customer issue

__Ticket Status & Resolution –__ Current state and final outcome of the ticket

__Ticket Priority –__ Priority level (Low, Medium, High, Critical)

__Ticket Channel –__ Mode of communication (Email, Phone, Chat, etc.)

__First Response Time –__ Time taken for initial response

__Time to Resolution –__ Total time taken to resolve the issue

__Customer Satisfaction Rating –__ Feedback score provided by the customer

This dataset enables performance evaluation at both operational and customer experience levels. It helps measure key metrics such as response efficiency, resolution effectiveness, SLA adherence, workload distribution, and overall customer satisfaction. The data supports analytical insights to improve service quality and optimize team productivity.
